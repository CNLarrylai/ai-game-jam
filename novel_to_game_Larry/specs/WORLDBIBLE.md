# 世界圣经规范(World Bible spec)—— 小说无关 · 可复用方法论

目标:把一本小说"贯穿全局、整个游戏都要复用的关键信息"一次性提炼成结构化的 `world_bible.json`。这是任何"小说→生存游戏"管线的**第一性产物**,先于、并服务于所有逐章卡点。它与逐章卡点是两类东西:
- **世界圣经** = 不变的底座(设定、人物、灾难逻辑、资源映射、基调)。一份。
- **逐章卡点** = 玩法实例(具体抉择)。很多。

## 为什么先做它
- 开篇章节(及序言)信息密度低却**沉淀世界观**——密度筛会把它们排在末尾,正好用来识别"哪里是设定,哪里是玩法"。
- 逐章提炼时需要"全局背景"喂给模型(此刻灾难到了哪一步);这份底座直接供给。
- 它是**跨小说复用**的:换一本书,流程不变,只是填充不同。

## 扫描 agent 要回报什么(供我汇总成最终 world_bible.json)
你只负责分配给你的若干章。**不要写卡点。** 读完后用结构化文字回报:
1. **世界圣经贡献**:针对上面每个字段,你这几章能补充的内容(没有就略过)。尤其注意 `setting`、`catastrophe.progression_phases`(你这几章处于哪个阶段)、`cast`、`setup_to_preserve`、`signature_quotes`。
2. **候选卡点时刻**:你这几章里**真正有"幸存者两难/张力"的时刻**,每个给:一句话描述 + **张力评分 1–5**(5=强抉择,1=纯抒情无抉择)+ 为什么。低于 3 分的不必勉强。我会跨整卷只挑最强的少数几个做卡,而不是每章都做。

## 文风
- 中文为主;`res` 等字段值用英文。

## 新增字段(v2:骨架 / 住所 / 路线 / POI)
在 `world_bible.json` 增产以下字段,供游戏壳按"住所 + 路线 + POI"通用框架渲染(见 GAME_FRAMEWORK.md §E):
```jsonc
"spine_type": "journey | holdout | social | investigation",   // 故事骨架(决定地图隐喻+住所是否移动)
"dwelling": {                          // "住所"轨迹(基地 = 人/载具,而非固定地点)
  "kind_progression": ["马车","温莎城堡","小船"],            // 随旅程演变
  "fragility_beats": ["暴雪夜马车散架→失伊德丽丝"]            // 受损/丢失的关键节点(即戏)
},
"route": [                             // 把章节聚成"程"
  { "leg": 1, "place": "温莎", "phase": "本土爆发", "chapters": ["V2-CVI","V2-CVII"] }
],
"poi_bank": [                          // 沿途据点(策展 + 留 AI 槽给观众生成)
  { "leg": 1, "name": "水闸破屋", "kind": "search|npc|hazard|shelter", "from": "novel|audience", "hook": "染疫弃者" }
]
```
判定 `spine_type`:多数可爬小说 = journey(地点在变);困守一地 = holdout(= cheney 的固定基地特例);群像/关系 = social;悬疑 = investigation。
