# 提炼引擎规范(distillation engine spec)

你的任务:把玛丽·雪莱《最后的人》(The Last Man)的**一个章节**,提炼成 1–2 个"末日生存叙事卡点"(currentNode),供一款直播互动叙事游戏使用。

## 输入
- 章节全文在 `data/chapters.json`,是一个数组,每个元素有 `id`、`text` 等字段。
- 你要处理的章节 `id` 会在调用你的指令里给出。读 `data/chapters.json`,按 `id` 找到你那一章的 `text`。
- (调用你时通常会直接给绝对路径,以它为准。)

## 你要做的事
从这一章里找出 **1–2 个"幸存者必须做艰难抉择"的时刻**,每个转化为一个卡点。
- **剥离抒情和景物描写**,只保留:发生了什么、谁要做决定、有什么取舍。
- **忠于原文**的事件和人物(主角第一人称叙事者是 Lionel Verney;核心人物有 Idris 伊德丽丝、Adrian 阿德里安、Perdita 佩迪塔、Raymond 雷蒙德等)。改编,不要另编一个故事。
- 如果这一章几乎全是抒情/铺垫、没有真正的抉择,就抓最接近"张力/两难"的一刻做成 1 个卡点即可;不要硬凑。

## 设计原则(必须遵守)
1. 每个卡点 **2–3 个真有取舍的选项**,不能有明显最优解。
2. 每个选项的代价落到**固定三种资源**上(`res` 只能是这三个英文值):
   - `food` 食物/水(生存底线)
   - `health` 健康/药品(伤病、感染)
   - `morale` 士气/人性(做泯灭人性的选择会掉)
   `d` 是整数增减,如 `-2`、`3`。
3. 每个卡点**至少一个选项** `risk: true`:它的精确 costs 和 outcome 是"选后揭晓"。risk 选项的 `hint` 要刻意模糊(给情绪、不给数值、不剧透结果)。
4. `effects` 标注这个选择对未来的因果影响(轻量文字桩即可,如「少了两份口粮 → 营地存粮更紧」)。可以 1–2 条。
5. `next` 一律填 `""`(跨章节的卡点串联由汇总阶段统一处理)。**例外**:如果你这一章产出了 2 个卡点且第一个自然导向第二个,可把第一个卡点里选项的 `next` 填成第二个卡点的 `id`。

## 输出格式(严格对齐)
写一个 **JSON 数组**(1–2 个 node)到 `build/cards/<你的章节id>.json`,例如 `build/cards/V2-CVI.json`。每个 node:

```json
{
  "id": "snake_case_场景名",
  "title": "短标题(决定的那一刻)",
  "desc": "2-3 句:当前处境 + 要做的决定",
  "generated": true,
  "icon": "单个emoji",
  "choices": [
    {
      "id": "snake_case_选项id",
      "label": "玩家选的动作",
      "hint": "选前的模糊暗示(risk选项尤其不给数值、不剧透)",
      "costs": [ {"res":"food","d":-2}, {"res":"morale","d":2} ],
      "risk": true,
      "outcome": { "emoji":"🤝", "title":"选后揭晓的标题", "body":"选后揭晓的后果文字" },
      "next": "",
      "effects": ["因果桩1","因果桩2"]
    }
  ]
}
```

## 文风约定
- **玩家可见文字**(`title`/`desc`/`label`/`hint`/`outcome.title`/`outcome.body`/`effects`)用**简体中文**,简洁、有末日生存的紧张感。
- **`id` 和 `res`** 用英文 snake_case。
- `icon`/`emoji` 各一个表情。

## 风格样例(已验收,照这个质感来)
节点「孤舍里的弃者」:逃难者把染疫同伴丢在破屋,村民拦着主角去救——救垂死陌生人 vs 保护家人。三个选项:
- 进屋照料他(`risk:true`,士气+2/健康−1;揭晓:人早已死,惊逃但未染病)
- 屋外留下水和干粮(食物−2/士气+1)
- 转身回家(士气−3)

## 收尾
- 写文件前自己确认 JSON 能被 `JSON.parse` 解析。
- 完成后,用一句话回报:章节 id、产出几个 node、各 node 的标题。
