# 废土风格规范（Wasteland Style Bible）

> 题材：AI 统治世界的末日废土生存。本规范为今晚批量生产六类资产的唯一执行标准。
> 所有 hex 为白名单，超出白名单的颜色一律不允许出现在最终资产里。
> 凡本文写"必须 / 禁止 / =N"的均为可质检硬指标，不接受"原则上""尽量"。

---

## 1. 风格一句话 + 三个参照锚点

**一句话**：精细像素 + 选择性描边 + 色相偏移的"冷暖温度对立"——用暖光代表幸存人类、冷青代表 AI 机器，一眼读出末日叙事。

三个参照锚点（只拆风格规律，不抄素材）：

| 锚点 | 借它的什么规律 |
|---|---|
| 星露谷物语（Stardew Valley） | 借它的"物资图标语言"：16px 小图标用 3-4 级明度 + 1px 深色外描边 + 左上单一高光点，远看也能秒辨物品类别。 |
| 泰拉瑞亚（Terraria） | 借它的"tile 无缝拼接 + 选择性描边"：地图 tile 四边像素对齐可平铺，物件外轮廓描边、内部色阶过渡不描边，避免"黑框糊一团"。 |
| Lospec 废土向成熟色板（Resurrect 64 / Endesga 32） | 借它的"成熟色坡关系"：明度梯度均匀、每级带色相偏移，不自己凭感觉调灰。本规范全局色板即从这两套抽取改造，见第 3 章。 |

---

## 2. 三档尺寸规范

光源、描边、色坡规则全档统一（见第 4 章），差异只在"细节密度上限"和"最少色数"。

| 档位 | 用途 | 画布 | 细节密度上限 | 色数范围 |
|---|---|---|---|---|
| 16px | 物资图标、小道具 | 16×16 | 最多 1 个高光点 + 1 个标识记号；不画分件结构，靠剪影辨识 | 最少 3 色，最多 6 色（含描边色） |
| 32px | 地图 tile、小型角色 | 32×32 | 最多 2 层结构（如角色"头+身"或 tile"地表+杂物"）；每结构 3 级色阶 | 最少 5 色，最多 12 色 |
| 48px | 主要角色立绘、同伴 | 48×48 | 最多 4 个可辨部件（头/躯干/装备/手持物）；面部只给 2-3 像素表情 | 最少 8 色，最多 16 色 |

补充硬规则：
- 三档画布尺寸固定，禁止非整数倍缩放交付（见第 4 章禁止项）。
- 48px 立绘的"有效绘制区"留 2px 安全边，避免贴边被 UI 裁切。

## 3. 全局调色板（核心产出）

废土总色板 **28 色**，分 5 组。所有资产只能用下列 hex，UI 中性灰额外白名单见组末。
来源标注：R64=Resurrect 64，E32=Endesga 32；"改造"指在源色基础上做了明度/饱和度微调以拉开色坡或制造温度对立。

色坡读法：每组从暗到亮（L1 最暗 → L5 最亮），并标注暗部色相偏移方向。

### A. 底色组 · 灰土 / 废墟混凝土（暗部偏紫灰）
> 偏移方向：暗部向紫灰偏（不要纯黑灰），最暗级直接借 R64 的暗紫底色。

| 级 | hex | 说明 |
|---|---|---|
| L1 | `2e222f` | 最暗，紫灰黑，全局通用阴影底 / 描边备用（R64） |
| L2 | `3e3546` | 暗混凝土，偏紫（R64） |
| L3 | `625565` | 中灰土，紫灰（R64） |
| L4 | `7f708a` | 亮灰，紫调（R64） |
| L5 | `9babb2` | 高光灰，微青（R64，作底色组顶高光） |

## 4. 像素纪律（全档强制）

**选择性描边（Selective Outline）**
- 外轮廓：用本物件主色组的 L1（最暗级）描 1px 外轮廓，不用纯黑。AI 物件外轮廓用 `0b5e65`，人类物件用 `2e222f`。
- 选择性提亮：物件左上受光区的外轮廓，把描边色提升一级（L1→L2），让受光边"亮起来"；右下背光区保持 L1。
- 内部色阶：物件内部不同色块之间禁止再加深色描边，靠相邻色阶自然过渡（避免"黑线切割糊成一团"）。
- 与负空间相接、且处于受光顶边时，可断开描边（留空）以求轻盈，仅限 32px/48px 档使用。

**统一光源**
- 全资产光源固定在左上 45°。高光永远在左上，最深阴影永远在右下。禁止逐资产乱打光。

**手动 AA（抗锯齿）**
- 仅手动加在"长曲线 / 大于 30° 的斜边"的阶梯转折处，用相邻两色的中间明度补 1 像素。
- 禁止在 1-2 像素的小图标边缘加 AA（会糊）。16px 档默认不做 AA。

**Dithering（抖动）**
- 只允许用于 32px/48px 的大面积材质过渡（如混凝土地表、雾霾天空、污水面），且必须是规则棋盘格抖动。
- UI 套件全面禁用 dithering。小图标禁用 dithering。

**禁止项（出现即打回）**
- 禁止自动缩放插值（nearest 以外的缩放）、禁止非整数倍缩放。
- 禁止半透明渐变 / alpha 渐变作为绘制手段（发光统一用纯色色阶台阶模拟）。
- 禁止使用第 3 章白名单以外的任何 hex。
- 禁止人类物件主色使用 C 组（AI 冷青）；禁止 AI 物件主色使用 D 组（暖光）。

## 5. 按资产类型规格表（今晚六类）

| 资产类型 | 尺寸档 | 色数上限 | 必含视觉记号 | 禁忌 |
|---|---|---|---|---|
| 物资图标（食物/水/药品/工具/武器/材料） | 16px | 6色 | 单一剪影可辨；左上1高光点；人类物资必带1处磨损/补丁像素（B组暗色点） | 禁止文字标签；禁止C组冷青作主色（除非该物资是AI拾取物）；禁止dithering |
| 角色与同伴（玩家+NPC） | 48px（小型NPC可32px） | 16色 | 头/躯干/装备/手持物四部件可辨；2-3像素面部表情；同伴需配色与玩家区分一冷一暖 | 禁止超4部件；禁止纯黑描边；待机帧位移>2像素 |
| 家/庇护所场景 | 32px tile拼 | 12色/tile | 至少1处D组暖光源（篝火/灯）；木板补丁/防水布破损细节 | 禁止出现C组冷青大面积（家=人类安全区）；禁止半透明灯光 |
| 地图 tile（探索网格） | 32px | 12色 | 四边像素对齐可无缝平铺；地表+0~1件杂物两层 | 边缘像素禁止贴满防接缝（留呼吸）；禁止四边像素不对称导致拼接错位 |
| 地点卡（选地图缩略卡） | 48px图标+边框 | 16色 | 一眼读出地点性质（AI设施=冷青主调，人类废墟=锈橙主调）；远景雾霾灰渐变背景 | 禁止细碎到看不清主体；禁止暖冷混用导致性质模糊 |
| UI 套件（按钮/面板/行动点/日计数/事件卡框） | 矢量像素，按9宫格切 | 仅用F组+少量强调色 | 按钮三态（常态/悬停/按下）明度各差1级；面板九宫格4角+4边可拉伸；行动点5格用D组暖黄=可用、F组3a4466=已用 | 禁止dithering；禁止圆角抗锯齿；禁止F组以外色做底（强调用B组警示红/C组AI青） |

**角色 2 帧待机可行性说明**：48px 角色做"2 帧待机"完全可行且低成本——只需复制基础帧，对"胸口呼吸区"做整体上移/下移 1 像素 + 手持物末端 1 像素摆动，每帧间隔 600-800ms 循环。不需要重画，工作量约为基础帧的 15%。同伴同理。32px 小型 NPC 可省略待机帧，用静态站立。

## 6. 题材视觉叙事三规则（写死，可质检）

> 这三条让整套资产"一眼末日 AI"。每条都给出客观验收标准。

1. **冷青发光律**：所有 AI 物件（无人机/扫描仪/全息/AI 设施/AI 拾取物）必带 1px 冷青发光描边，发光色固定 `30e1b9`（边缘）或 `8ff8e2`（最亮点）。验收：随机抽 AI 资产，外轮廓必须能找到至少一段冷青提亮描边；人类物件不得出现此发光。

2. **人类磨损律**：所有人类物件（物资/装备/家具/废墟）必含可见磨损细节——补丁、锈点、裂纹或破口，至少 2 处用 B 组暗色（`6e2727`/`b33831`）点缀。验收：抽人类资产，必须能数出 ≥2 处磨损像素；干净无瑕的人类物件一律打回。

3. **雾霾远景律**：任何带背景/远景的资产（地点卡、场景、地图远层）天空与远景永远是雾霾灰渐变档——顶部 `625565` → 底部 `9babb2` 的纵向台阶渐变（纯色阶，非 alpha），中间不得出现纯蓝晴空。验收：检查背景顶到底是否为底色组 A 的灰阶台阶；出现明亮蓝天/纯黑夜空均打回。

## 7. HD-2D 后期增强层（可开关，演示加分项）

整层叠加在像素渲染层之上，CSS filter / overlay 实现，一个开关全控，演示时打开、出问题可秒关回纯像素。

```css
/* 演示增强层：套在 #game-canvas 外层容器 */
.hd2d-enhance {
  filter: contrast(1.06) saturate(1.08) brightness(1.02);
  image-rendering: pixelated; /* 像素层本体保持硬边，不被 filter 糊 */
}

/* 暗角 vignette：聚焦中心，强化末日压抑 */
.hd2d-enhance::after {
  content: ""; position: absolute; inset: 0; pointer-events: none;
  background: radial-gradient(ellipse at center,
    rgba(0,0,0,0) 55%, rgba(46,34,47,0.55) 100%); /* 用底色2e222f 调暗角 */
}

/* bloom 辉光：仅对暖光/冷青高光区轻微外溢，用 mix-blend screen 叠一层模糊高光图 */
.hd2d-bloom {
  mix-blend-mode: screen; filter: blur(2px); opacity: 0.35;
}

/* 扫描线：强化"被 AI 监视"的电子质感，可单独开关 */
.hd2d-scanline::before {
  content: ""; position: absolute; inset: 0; pointer-events: none;
  background: repeating-linear-gradient(
    to bottom,
    rgba(48,225,185,0.00) 0px,
    rgba(48,225,185,0.00) 2px,
    rgba(11,94,101,0.10) 3px); /* 冷青 30e1b9 / 0b5e65 极淡扫描线 */
  opacity: 0.5;
}
```

注意：HD-2D 层是"演示糖"，关掉后纯像素资产必须仍然成立（自洽达标）。不允许靠后期层去补救资产本体的色板/描边问题。

## 附：拍板记录（2026-06-04 林子尧）

1. **同伴用色**：C 组冷青为 AI 阵营独占。同伴的冷色调一律用 E 组污绿或 A 组紫灰，禁碰 C 组。
2. **AI 物资图标**：物资图标主色一律暖/锈色系；AI 来源物资用 **C 组青色 1px 边框**区分归属，主色不变。
3. **立绘色数**：48px 立绘维持单张 ≤16 色。表现力不足时可申请放宽至 20 色（需美术总监批准，不得自行放宽）。

> 以上三条在用户看到第一批真实资产时保留反悔权；如改判，只重跑受影响资产。

## 附：拍板记录 2（2026-06-05 林子尧，范例校准后）

4. **信息密集 tile 色数**：多材质满载 tile（如满货架）放宽至 ≤18 色；普通 tile 仍 ≤16。
5. **16px 图标色数**：≤9 色（覆盖第 4 章原"6 色"条款），以落实范例的明度坡 + 冷反光技法。
6. **生产方式**：一律临摹式生产（knowledge/pixel-craft-playbook.md §0：先 Read 指定范例、并排对照 ≥2 轮）。盲画产出质检直接打回。
